Método de aceptación y rechazo¶

El mtodo de la trasnformada inversa, permite generar datos que distribuyan como una funcin $f(x)$. Para poder aplicar este mtodo $f(x)$ debe de ser una funcin acotada y con un dominio finito $a\leq x \leq b$

De acuerdo a sta funcin $f(x)$ el mtodo propone los siguientes pasos. Asuma que podemos encontrar una funcin $t(x)$ tal que $$t(x)\geq f(x), \quad \forall x$$ Para este caso utilizamos $t(x)$ como el valor mximo de $f(x)$.

La funcin $t(x)\geq 0$ no es una PDF debido a $$\int_{-\infty}^{\infty}t(x)dx\geq \int_{-\infty}^{\infty}f(x)dx =1$$ Hacemos $T(x) = c$ $$c=\int_{-\infty}^{\infty}t(x)\geq 1$$ Definamos la funcin $g(x)=t(x)/c \rightarrow g(x)$ es una densidad.

Resultando entonces $$\frac{f(x)}{g(x)}\leq c,\quad \forall x$$ El siguiente algoritmo genera una variable aleatoria $X$, distribuida de acuerdo a la densidad $f(x)$

  1. Generar $R_1$ teniendo densidad $g(x)$
  2. Generar $R_2 \rightarrow U \sim U(0,1)$ independiente de $R_1$ del paso 1 .
  3. Evaluar la funcin de probabilidad en $R_1$.
  4. Determinar si la siguiente desigualdad se cumple: $$R_2\leq \frac{f(R_1)}{t(R_1)}$$ Si la respuesta es afirmativa se utiliza $X=R_1$, de lo contrario es necesario pasar nuevamente al paso 1, tantas veces como sea necesario. De esta forma podemos generar una cantidad $n$ de nmeros a partir de una distribucin $f(x)$

Kernel Density Estimation¶

Kernerl density estimation (KDE) es una tcnica estadstica no paramtrica, que nos perimite estimar la funcin de densidad de probabilidad de una variable. A grandes rasgos sta tcinca genera una lnea suave y continua, que se ajusta a la distribucin de los datos (finitos). Para hacer sto, la tcnica de KDE genera lneas a partir de los datos y se va ajustando a ellos de acuerdo a la probabilidad de ocurrencia, es decir, entre mayor presencia de datos tengamos dentro de una clase KDE, infiere que existe una mayor probabilidad de ocurrrencia de sta clase y por lo tanto va ajustando la funcin de acuerdo a sto. Una parte muy importante de KDE es el kernel, a travs del parmetro bandwidth (bw), es que cambia el kernel, cuando bw es bajo, se le dar ms peso a aquellos puntos que se encuentren cerca de la media, cuando bw es un valor, comienza a crecer, sto quiere decir que se le comienza a dar ms peso a los valores que se encuentran ms alejados de la media.

La forma en la que se calcula sta lnea o funcin que genera KDE, es a travs de la siguiente funcin: $$ \hat{ f}(x)= \sum_{observaciones} K(\frac{x- observaciones}{bandwith}) $$ Donde K representa el kernel, el kernel hace referencia a la funcin de distribucin que se aproxima, puede ser uniforme, guassiana, triangular, etc. En este caso utilizamos el mtodo de cross validation maximum likelihood (MLCV), sta se define de la siguiente forma (MLCV) $$MLCV(h) = n^{-1} \sum_{i=1}^{n} \log\left[\hat{f}_{h,i}(x)\right]$$

Funciones para el entrenamiento y prueba del modelo¶

Funcin de costo: $J(w)=-\frac{1}{m} \sum_{i=1}^{m} \big( y_i\ log(\sigma(z_i)) + (1-y_i)\ log(1-\sigma(z_i)) \big) + \frac{\lambda}{2m}\sum_{j=1}^n w_j^2$

Gradiente: $\frac{\partial J(w)}{\partial w_j} = \frac{1}{m} \sum_{i=1}^{m}(\sigma(z_i) - y_i) * x_i^j + \frac{\lambda}{m}w_j$

En su forma vectorizada: $\nabla J(w)=\frac{1}{m} X^T (\sigma(z) - y) + \frac{\lambda w}{m}$

Sigmoide: $\sigma(z_i) = \frac{1}{1+e^{-z_i}}$ where $z_i=x_iw^T$

Actiaulizacin de pesos:

  • $w_0 = w_0 - \alpha \frac{1}{m}\sum_{i=1}^{m}(\sigma(z_i) - y_i)$
  • $w_j = w_j - \alpha \frac{1}{m}\sum_{i=1}^{m}(\sigma(z_i) - y_i) * x_i^j + \frac{\lambda}{m}w_j$ para $j>0$

Medidas de desempeño¶

Una vez hecha la prediccin con nuestro modelo necesitamos saber qu tan bien o mal se desempea el mismo, para sto vamos a utilizar dos medidas de desempeo:

  • Precisin: sta medida nos indica la proporcin de datos que fueron clasificados correctamente
  • sensibilidad: sta medida nos indica la proporcin de 1, que son correctamente indentificados
  • especifidad: sta medida nos indica la proporcin de 0, que son correctamente identificados

Para calcular stas medidas, necesitamos conocer la cantidad de verdaderos positivos (TP), verdaderos negativos (TN), falsos positivos (FP) y falsos negativos (FN). Una vez calculados stos valores, aplicamos la frmula para precisin ($a_c$), sensibilidad ($s_e$) y especifidad ($s_p$).

$$a_c = \frac{TP+ TN}{TP+ TN + FP + FN}$$$$s_e = \frac{TP}{TP + FN}$$$$s_p = \frac{TN}{TN + FP}$$
